## 1 Введение
Данный документ описывает план тестирования мобильного приложения "CoinKeeper", предназначенного для учета личных финансов. Документ предназначен для команды тестирования. Цель тестирования - проверка соответствия реального поведения приложения ожидаемому поведению, описанному в требованиях, а также оценка его удобства и надежности.

## 2 Объект тестирования
В качестве объектов тестирования можно выделить следующие функциональные требования:

**Основные функции приложения:**
- Добавление операции (доход/расход)
- Категоризация операций
- Просмотр истории операций
- Статистика и аналитика
- Установка лимитов бюджета
- Экспорт данных
- Резервное копирование и восстановление
- Управление категориями

**Ключевые экраны приложения:**
- Экран главного меню
- Экран добавления операции
- Экран статистики
- Экран настроек

## 3 Атрибуты качества

### Функциональность:
- **Функциональная полнота:** приложение должно выполнять все заявленные функции
- **Функциональная корректность:** приложение должно выполнять все вычисления (баланс, статистика, лимиты) без ошибок

### Удобство использования:
- **Интуитивность интерфейса:** минималистичный дизайн, фокус на быстром добавлении операций
- **Соответствие гайдлайнам платформ:** Android/iOS

### Производительность:
- **Время отклика:** на действия пользователя < 100 мс

### Надежность:
- **Стабильность работы:** Crash-free rate ≥ 99.5%
- **Целостность данных:** сохранность финансовой информации пользователя

### Совместимость:
- **Кроссплатформенность:** корректная работа на различных версиях ОС (Android 8.0+ / iOS 13+) и размерах экранов

## 4 Риски
К рискам можно отнести:

- Потеря или повреждение пользовательских финансовых данных при обновлении приложения или сбое устройства
- Некорректный расчет баланса и статистики при большом количестве операций
- Проблемы с производительностью при длительной работе и накоплении обширной истории операций
- Несовместимость файлов экспорта/импорта после обновления формата данных
- Неудовлетворительная работа в оффлайн-режиме

## 5 Аспекты тестирования

### Функциональное тестирование:

#### 1. Добавление операции (доход/расход)
- Успешное добавление операции с обязательными полями (сумма, категория, тип)
- Добавление операции с необязательными полями (комментарий, дата)
- Попытка добавить операцию с некорректной суммой (ноль, отрицательное число, нечисловые символы)
- Проверка автоматического обновления текущего баланса на главном экране после добавления операции

#### 2. Категоризация операций
- Создание новой пользовательской категории
- Редактирование существующей категории
- Удаление неиспользуемой категории
- Попытка удалить категорию, к которой привязаны существующие операции
- Корректное отнесение операции к выбранной категории

#### 3. Просмотр истории операций
- Отображение списка всех операций в хронологическом порядке
- Фильтрация операций по типу (доход/расход), категории, периоду
- Поиск операции по комментарию
- Корректность отображаемых данных (сумма, категория, дата) для каждой операции

#### 4. Статистика и аналитика
- Корректное построение круговой диаграммы распределения расходов по категориям
- Корректное отображение графиков динамики доходов/расходов за разные периоды (неделя, месяц, год)
- Проверка расчета процентов для каждой категории в статистике
- Отображение статистики при отсутствии операций

#### 5. Установка и контроль лимитов бюджета
- Установка месячного лимита на категорию или общего лимита
- Проверка уведомлений/визуальных индикаторов при приближении к лимиту и его превышении
- Корректный расчет процента от лимита в статистике

#### 6. Резервное копирование и восстановление
- Создание резервной копии данных
- Восстановление данных из резервной копии
- Проверка целостности данных после восстановления
- Обработка попытки восстановления из битого файла

### Нефункциональное тестирование:

#### 1. Тестирование удобства использования (Usability)
- Интуитивность навигации между экранами
- Скорость и простота процесса добавления новой операции (количество тапов)
- Читаемость текста и элементов интерфейса
- Удобство восприятия графиков и диаграмм

#### 2. Тестирование производительности (Performance)
- Время запуска приложения (Cold start)
- Время отклика при добавлении, редактировании, удалении операций
- Скорость загрузки и отрисовки экрана статистики при большом количестве операций (1000+)
- Потребление памяти и заряда батареи

#### 3. Тестирование надежности (Reliability)
- Работа приложения в условиях нестабильного соединения
- Поведение приложения при нехватке памяти на устройстве
- Восстановление после принудительного завершения работы ОС

#### 4. Тестирование совместимости (Compatibility)
- Корректное отображение и работа на разных версиях ОС (Android 8.0+ / iOS 13+)
- Корректное отображение на разных разрешениях экрана и плотностях пикселей
- Корректная работа при смене ориентации устройства

#### 5. Тестирование доступности (Accessibility)
- Корректное масштабирование интерфейса при использовании системного крупного шрифта
- Достаточный цветовой контраст для элементов интерфейса
- Корректная работа с VoiceOver/TalkBack

## 6 Подходы к тестированию
При тестировании будет использован ручной подход. Для регрессионного тестирования ключевых функций будет рассмотрена возможность создания и автоматизации smoke-тестов.

## 7 Представление результатов
Результаты тестирования будут представлены в документе "Результаты тестирования", который будет включать:

- Общую информацию о ходе тестирования
- Процент успешно пройденных тест-кейсов
- Список обнаруженных дефектов с указанием критичности, шагов для воспроизведения и ожидаемого/фактического результата
- Логи и скриншоты для наглядной демонстрации дефектов
- Рекомендации по исправлению дефектов и дальнейшему тестированию

## 8 Выводы
Данный тестовый план позволяет протестировать основной функционал мобильного приложения "CoinKeeper". Успешное прохождение всех тестов позволяет с высокой долей уверенности полагать, что приложение работает корректно, соответствует заявленным функциональным и нефункциональным требованиям и готово к выпуску.
